Doodle dream School invasion

Doodle dream school invasion is a top-down shooter where you have to kill monsters in order to escape the school. At the beginning of the game, you get to choose which weapon you want to use to kill the monsters. In order to progress, you have to kill all the monsters in the room and shoot down the door to go to the next room. Once you've found the room with the stairs, you'll be able to go down a floor. You can find the exit of the school on the main floor but be careful because there is a terrible monster guarding it.

Breakable door

We wanted more action in our game. So we thought it was a fun idea to shoot down the doors instead of opening a door. For this to work, I made an array with all the rigid body's of the door pieces and then gave the door some health. When a bullet collides with the door, the hp of the door goes down. When the hp is down to zero, a foreach loop will add a force, a direction and a random torque to all the rigidbody's of the door pieces.

Room detection

We had a problem where the enemies were able to immediately go towards the player when the game began. This was an issue because we don't want enemy's to attack the player when the player is not in the same room as the enemy's. To fix this, I made a room detection. This made sure that the enemy's only started moving when the player was in the same room as the enemy's and it also turned on the lights to show that the room got activated.

The gun

For the gun, we wanted to be able to switch between multiple guns to add a bit of variety when killing the monsters. In order to get this working, I made sure the guns were scriptable objects with a fire and reload function. This gave me the ability to make a script where I could load in the gun and have easy access to its functions.

Make me laugh

Make me laugh is the first game jam I participated in. The theme was make me laugh. We were in a group of 8 people and we had two days to make a game. The game is inspired by a game by the name of shotgun roulette where you play shotgun roulette with an eldritch monster. Your objective is to use various items to make sure the Eldritch monster shoots himself. We wanted to make something similar, where you had cards with jokes on them to make a funny joke.

Generating the cards

We wanted the cards to be 3D objects and to make sure that it was easy to change around its values. To make this happen, I made scriptable objects of the cards that can hold the information of the card. These cards are put into a list and card objects get made for every scriptable object in the list. Once they were created they were removed from the list.

Cards moving to your hand

We thought it would be cool to make the cards move from a deck to the players hand. I accomplished this by having three empty objects in an array and then move the cards to their position with a for loop.

  • Review code

Selecting the cards

We also needed to have a way to select the card to make the joke. I accomplished this by having them move up when the mouse hovered over a card. This makes it easier to see which card you're going to select and what the card says. If you then select the card that you want to use, a value would be checked called funnyModifier to see if the card if the joke is funny or not. If the value is +1, the joke is considered funny, 0 if the joke was not that funny but not that bad either, and -1 if the joke was bad.

AudioManager

We also needed some audio to set the mood. For this, I made an array of Sounds. These sounds had some values, you could change. Then in the audio manager script new audio source components are made from those values. Lastly the audio manager had a public function called play, which took in a string. If wanted to play a certain sound at some point I can call up the instance of the audio manager and say which sound needs to be played by typing the name of the sound.

SteamSky

SteamSky is my first project where I worked together with another game developer and game artists. The game is a bullet hell side scroller game. In this game, you have to dodge bullets from enemies that move towards you in unique patterns. Score as many points as possible and defeat the final boss at the end.

The wave spawner

When we were discussing how we wanted to spawn enemies, we came to the conclusion that We wanted the waves of enemies to be fully customizable so that every wave looked unique. So for every wave you can give it a name, add enemies to a list and per enemy you can say what their movement is supposed to be. You can choose between move straight, move along a sin wave or a custom path.

Highscore

We wanted to add high scores to encourage players to aim for the best scores and compete with their friends. It works by asking the player at the win screen to submit a name. This name and score saved in the playerprefs and converted to a JSON file. When the player goes to high score screen the high scores get loaded from the JSON file and are ordered based on highest score.

The Platformer

This is my first game that I have ever made. It was made in an engine called Canvas Engine, which was an engine to help us learn to code before we started on working in Unity. It was really challenging because I wanted to add a lot of things that Unity has implemented that I needed to make myself.

Velocity

The player needed some way of moment. That's why I sort of made velocity. For the left and right movement, I had a value that would increase when moving right and decrease when moving left. For the y movement, I had a constant value that would move the y of the character down. When the player jumps the y value gets increased over time and then decreases to simulate gravity.

Collision detection

Without collision, you can't play a game. What I did to achieve this was check if the x position and y position of the player plus the player's width and height and check if they were inside the x and y plus width and height of the object that needed collision. If it was, then the gravity of the player would be set to zero.

Spikes

I wanted to add some variety to make the game not boring. To make this happen I added moving and static spikes. If the player collided with one, the player would lose and had to restart the level.
