Doodle dream School invasion

Doodle dream school invasion is a top-down shooter where you have to kill monsters in order to escape the school. At the beginning of the game, you get to choose which weapon you want to use to get out of the building. In order to progress, you have to defeat all the monsters in the room and shoot down the doors to enter the next room. Once you've the stairs, you'll be able to go down a floor. You can find the exit of the school on the main floor but be careful because there is a terrible monster guarding it.

Breakable door

We wanted more action in our game. So we thought it was a fun idea to shoot down the doors instead of opening them. The problem was that the door had multiple pieces that had to fly away in order to make it look like it was getting blown up. To fix this problem, I made an array with all the rigid body's of the door pieces and then gave the door some health. When a bullet collides with the door, the hit points of the door goes down. When the health of the door is down to zero, a foreach loop adds force, direction and a random torque to all the rigidbody's of the door pieces to make it look like it is exploding.

Room detection

We had a problem where the enemies were able to immediately go towards the player when the game began. This was an issue because we don't want enemy's to walk towards the player and/or attack it when the player is not in the same room as the enemy's. In order to get rid of this problem, I made a room detection. This made sure that the enemy's only started moving when the player was in the same room as the enemy's. It also turned on the lights in the room to indicate that this the room was "active".

The gun

For the gun, we wanted to be able to switch between multiple guns to add variety to our game when fighting the monsters. In order to get this to work, I made the guns scriptable objects with some simple variables and reload- and shoot function. With another script I could then make a serialize field where I could fill in the gun based on what the weapon the player chooses. In the same script I would then check for input from the player that would call the reload and shoot functions when the shoot or reload buttons were pressed.

You play as a cat that wants to get to the bottom of a well where a delicious snack is waiting to be eaten. During your fall, you encounter monsters that don't want you there and try to close in on you and attack you.

UIController

For the controls of the UI, we wanted to use the buttons that were on the screen to select an option. This came with a problem because you can't really select a button with a button. To solve this issue, I put the buttons of the menu in an array. This creates an order the buttons are in. The order would be kept track of with an integer. When the integer would increase or decrease by pressing the left or right button, the button in the menu would turn yellow based on the order they were in in the array and what number the integer is. To select an option from the menu you can press the jump button. This would call the function of the selected button.

Make me laugh

Make me laugh is the first game jam I participated in. The theme was make me laugh. We were in a group of eight people and we had two days to make a game. The game is inspired by a game by the name of shotgun roulette where you play shotgun roulette with an eldritch monster. Your objective is to use various items to make sure the Eldritch monster shoots himself. We wanted to make something similar, where you had cards with jokes on them to make a funny joke.

Generating the cards

We wanted the cards to be 3D objects to fit with the 3D theme of the game. To make this happen, I made scriptable objects of the cards that can hold the information of the card. These cards are put into a list and card objects get made for every scriptable object in the list. Once they were created they were removed from the list to show so that there wouldnt be an infinate amount of cards being generated.

Cards moving to your hand

We thought it would be cool to make the cards move from a deck of cards to the hands of the player. I accomplished this by having three empty objects in the scene and put them in an array. Then when the game starts or the player would get a new set of cards in his hands, the cards would move towards the positions of the empty objects with a for loop to keep track of which card would move to which empty object.

  • Review code

Selecting the cards

We also needed to have a way to select the card to make the joke. I accomplished this by moving the cards up when the mouse hovered over a card to indicate what card the player would be selecting. If you then select the card that you want to use, a value would be checked called the funnyModifier to see if the joke on the card is funny or not. If the value is +1, the joke is considered funny, 0 if the joke was not that funny but not that bad either, and -1 if the joke was bad.

AudioManager

We also needed some audio to set the mood. For this, I made a different class called sounds and made an array of it. an array of Sounds. These sounds had some values, you could change. SteamSky

SteamSky is my first project where I worked together with another game developer and game artists. The game is a bullet hell side scroller game. In this game, you have to dodge bullets from enemies that move towards you in unique patterns. Score as many points as possible and defeat the final boss at the end.

The wave spawner

When we were discussing how we wanted to spawn enemies, we came to the conclusion that We wanted the waves of enemies to be fully customizable so that every wave looked unique. So for every wave you can give it a name, add enemies to a list and per enemy you can say what their movement is supposed to be. You can choose between move straight, move along a sin wave or a custom path.

Highscore

The Platformer

This is my first game that I have ever made. It was made in an engine called Canvas Engine, which was an engine to help us learn to code before we started on working in Unity. It was really challenging because I wanted to add a lot of things that Unity has implemented that I needed to make myself.

Velocity

The player needed some way of moment. That's why I sort of made velocity. For the left and right movement, I had a value that would increase when moving right and decrease when moving left. For the y movement, I had a constant value that would move the y of the character down. When the player jumps the y value gets increased over time and then decreases to simulate gravity.

Collision detection

Without collision, you can't play a game. What I did to achieve this was check if the x position and y position of the player plus the player's width and height and check if they were inside the x and y plus width and height of the object that needed collision. If it was, then the gravity of the player would be set to zero.

Spikes

I wanted to add some variety to make the game not boring. To make this happen I added moving and static spikes. If the player collided with one, the player would lose and had to restart the level.
