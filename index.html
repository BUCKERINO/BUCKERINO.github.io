Doodle dream School invasion

Doodle dream School invasion is a topdown shooter where you have to kill monsters in order to escape the school. At the beginning of the game you get to choose which weapon you want to use to kill the monsters. In order to progress you have to kill all the monsters in the room and shoot down the door to go to the next room. Once you've found the room with the stairs you'll be able to go down a floor. You can find the exit of the school on the main floor but be careful because there is terible monster gaurding it.

Breakable door

We wanted more action in our game. So we thought it was a fun idea to shoot down the doors instead of opening a door. For this to I made an array with all the rigidbody's of the door pieces and the gave the door some health. When a bullet collides with the door the hp of the door goes down. When the hp is down to zero, a foreach loop will add a force, a direction and a random torque to all the rigidbody's of the door pieces.

Room detection

We had a problem where the enemies were able to immidiatly go towards the player when the game began. This was an issue because we dont want enemy's to attack the player when the player is not in the same room as the enemy's. To fix this I made a room detection. This made sure that the enemy's only started moving when the player was in the same room as the enemy's and it also turned on the lights to show that the room got activated.

The gun

For the gun we wanted to be able to switch between multiple guns to add a bit of variety when killing the monsters. In order to get this working I made sure the guns were scriptable objects with a fire and reload function. This gave me the ability to make a script where I could load in the gun and have easy acces to its functions.

For this project me and my brother worked together to make Nekolow. This was interesting because he had no experience in game development whatsoever. But that didn't stop us. The game is inspired by a game called downwell. You play as a cat that wants to get to the bottom of the well where a delicious snack is waiting for it. During your fall you encounter monsters that dont want you there and try to attack you. You kill them by shooting them or jumping on them.

UIController

For the control of the UI we wanted to use buttons on the screen to select your options. This came with a complication because you cant select a button with another button. To solve this issue I put the menu buttons in an array. When left or right is pressed. an interterger would increase and the color of the button would change based what number the interger is. When you press the jump button the invoke of the selected button gets called.

Weapon Manager

We wanted to make different weapons that you are able to pick up. To make this happen I made a weapon manager script that manages what weapon is currently used and the what that shoot property should be. If the player would shoot the shoot method from the weapon script would be called. This script uses an interface called IProjectile so that it would be easier to make more weapons if wanted more.

SteamSky

SteamSky is my first project where I worked together with another game developer and game artists. The game is a bullet hell side scroller game. In this game you have to dodge bullets from enemies that move towards you in unique patterns. score as many points as possible and defeat the final boss at the end.

The wave spawner

When were discussing how we wanted to spawn enemies we came to the conclusion that We wanted the waves of enemies to be fully customizable so that every wave looked unique. So for every wave you can give it a name, add enemies to a list and per enemy you can say what their movement is supposed to be. You can choose between move straight, move along a sin wave or a custom path.

Highscore

We wanted to add highscores to encourage players to aim for the best scores and compote with their friends. It works by asking the player at the win screen to submit a name. This name and score saved in the playerprefs and converted to a jason file. When the player goes to highscore screen the highscores get loaded from the jason file and are ordered based on highest score.
